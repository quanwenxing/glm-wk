# 開発プロジェクト管理ガイド

## GitHub Projects セットアップ手順

### 1. GitHub Projects ボードの作成

1. GitHub リポジトリの「Projects」タブをクリック
2. 「New project」をクリック
3. 「Board」テンプレートを選択
4. プロジェクト名を設定（例: 「MVP開発」）
5. 以下のカラムを作成:
   - `Backlog` - 未着手のタスク
   - `To Do` - 今週〜今スプリントで実施するタスク
   - `In Progress` - 現在実施中のタスク
   - `In Review` - レビュー待ちのPR
   - `Done` - 完了したタスク

### 2. ラベルの作成

以下のラベルをリポジトリに作成:

| ラベル名 | 色 | 説明 |
|----------|------|------|
| `sprint-1` | #e99695 | スプリント1（基盤構築） |
| `sprint-2` | #bfd4f2 | スプリント2（コンポーネント・API開発） |
| `sprint-3` | #d4a5a9 | スプリント3（ページ実装） |

### 3. Issues の作成

要件定義書のタスク一覧（セクション12）を参考に、各タスクをIssueとして作成。

**Issueタイトル形式**: `[タスクID] タスク名`
**例**: `[A-1] Next.jsプロジェクト初期化`

## 開発フロー

### 1. タスク割り当てフロー

```
Backlog → To Do → In Progress → In Review → Done
```

1. **Backlog**: 全タスクを最初に登録
2. **To Do**: 統括管理者が各担当者にタスクを割り当て
3. **In Progress**: 担当者が作業開始時にステータス変更
4. **In Review**: PR作成時にステータス変更
5. **Done**: レビュー完了・マージ後にステータス変更

### 2. ブランチ戦略

```
main (本番環境)
  └── feature/機能名
```

- **main**: 本番環境、常にデプロイ可能な状態
- **feature/機能名**: 各機能開発用ブランチ
  - 例: `feature/theme-list`, `feature/quiz-page`

### 3. 開発手順

1. **Issueの確認**
   - 割り当てられたIssueの詳細を確認
   - 依存タスクがある場合は、それらが完了しているか確認

2. **ブランチの作成**
   ```bash
   git checkout main
   git pull origin main
   git checkout -b feature/機能名
   ```

3. **開発実施**
   - 機能を実装
   - コミットメッセージ: `[タスクID] 簡単な説明`

4. **テスト**
   - ローカルで動作確認
   - 既存機能への影響を確認

5. **プルリクエストの作成**
   - PRタイトル: `[タスクID] タスク名`
   - PRテンプレートに従って記入
   - 関連Issueを `Closes #XX` で紐付け

6. **コードレビュー**
   - 統括管理者または指名担当者がレビュー
   - 指摘事項があれば修正

7. **マージ**
   - レビュー承認後にmainブランチへマージ
   - Squash and merge 推奨

8. **Issueのクローズ**
   - マージ後、Issueを `Done` に移動

### 4. コミット規約

```
[タスクID] タイプ: 簡単な説明

[詳細な説明があれば]

Refs: #issue番号
```

**タイプ例**:
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント更新
- `style`: フォーマット
- `refactor`: リファクタリング
- `test`: テスト追加
- `chore`: その他

**例**:
```
[F-1] feat: ホームページの基本レイアウト実装

科目・学年選択のセクションと、テーマカードの一覧表示を実装

Refs: #12
```

## 進捗管理

### 朝のスタンドアップ（15分）

各担当者が以下を共有:

1. **昨日の成果**: 何を完了したか
2. **今日の予定**: 何をする予定か
3. **ブロッキング**: 困っていることがあれば共有

### 週次進捗会議（1時間）

1. 今週の成果を振り返り
2. 来週の計画を共有
3. リスク・課題を議論
4. マイルストーンの達成状況を確認

### 統括管理者の確認項目

- [ ] GitHub Projectsで全タスクが可視化されている
- [ ] 各担当者の作業負荷が適切に分散されている
- [ ] 依存関係が明確で、ブロッキングが発生しないよう調整されている
- [ ] PRが timely にレビューされている
- [ ] CI/CDが正常に動作している
- [ ] バグや懸念点がIssue化され対応されている

## トラブルシューティング

### ブロッキングが発生した場合

1. すぐにIssueを作成して状況を共有
2. スタンドアップで統括管理者に報告
3. 統括管理者がサポートを調整

### タスク遅延が発生した場合

1. 速やかに統括管理者に報告
2. 遅延の原因と対応策を検討
3. 必要に応じて優先度調整

### コンフリクトが発生した場合

1. 定期的にmainブランチへ同期
2. 機能ブランチを小さく保つ（1タスク1ブランチ推奨）
3. コンフリクト解決後は必ずテストを実施
